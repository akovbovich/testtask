#### Input
```
1. На вход и на выход подаётся не список, а на вход стрим и на выход стрим. Тики во входящем стриме упорядочены по времени.
2. Как правило, свечка отдаётся, когда приходит тик со временем следующей свечи. Но у этого подхода есть одна проблема: если у тебя перерыв в расписании с 10 вечера до 8 утра и в этот интервал новых тиков не поступает, и ты отдаешь часовые свечки, то свечку за 10 вечера ты отдашь только в 8 утра. Нужно решить эту проблему и отдавать свечку вовремя.
3. Программа должна работать в риалтайме.
4. Реализовывать проект с использованием akka streams/zio/cats effect на выбор.
```
